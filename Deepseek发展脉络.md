# Deepseek发展脉络

**第一阶段：筑基与探索 (2024年初)**

*   **基础框架**：以 **DeepSeek LLM (2401.02954)** 为起点，建立了7B/67B的Dense模型基线，同时深入研究了Scaling Laws，为后续模型和数据规模的扩展提供了理论指导。
*   **核心架构创新**：
    *   **DeepSeekMoE (2401.06066)**：针对传统MoE的“知识混杂”和“知识冗余”问题，提出**细粒度专家分割**和**共享专家隔离**两大策略，旨在实现“终极专家专业化”。这成为了后续所有DeepSeek大模型的基础架构。
    *   **DeepSeekMath (2402.03300)**：展示了领域模型潜力，并通过GRPO算法初步探索了强化学习在提升推理能力上的作用。

**第二阶段：架构融合与效率革命 (2024年中 - 2025年初)**

*   **架构集大成者**：**DeepSeek-V2 (2405.04434)** 首次将**DeepSeekMoE**和为解决推理效率而生的**Multi-head Latent Attention (MLA)** 集于一身，实现了“更强性能、更省训练、更快推理”的三角突破。这是DeepSeek技术体系成熟的标志。
*   **规模化与工程优化**：**DeepSeek-V3 (2412.19437)** 将V2的架构规模推至671B参数，并引入**辅助损失免费的负载均衡策略**和**多Token预测（MTP）** 目标。在工程上，通过**FP8混合精度训练**和极致的通信优化，实现了极低成本的超大规模训练，证明了技术路线的商业可行性。

**第三阶段：推理能力爆发与深度优化 (2025年至今)**

*   **纯强化学习的推理涌现**：**DeepSeek-R1 (2501.12948)** 证明了即使在预训练后不使用监督微调（SFT），仅通过**大规模强化学习**也能让模型涌现出强大的推理能力（如自我反思、验证）。
*   **注意力机制新范式**：**DeepSeek-V3.2 (2512.02556)** 和 **DeepSeek-V4** 引入了**DeepSeek Sparse Attention (DSA)** 及更复杂的混合注意力架构，**从算法层面攻克长上下文计算瓶颈**，大幅降低推理计算量和KV缓存，使百万级Token上下文支持成为现实。
*   **记忆与连接的革新**：**Engram (2601.07372)** 探索了**条件记忆**作为MoE的补充，通过稀疏查表分担静态知识检索，让计算更专注于动态推理。**mHC (2512.24880)** 则从**拓扑结构**入手，通过为残差连接施加流形约束，在提升模型深度的同时保证了训练稳定性。
*   **系统架构终极优化**：**DualPath (2602.21548)** 深入推理服务底层，利用**双路径KV缓存加载**和**CNIC中心化流量管理**，打破存储I/O瓶颈，实现高效的PD分离式推理。

### 📜 详细发展脉络梳理

以下是基于论文的详细、逐步的解读。

#### 1. 奠基：开创性的架构双星

在早期探索后，DeepSeek团队确立了以Mixture-of-Experts (MoE)为核心的技术路线，并通过两篇论文分别解决了MoE的性能上限和推理效率问题。

*   **DeepSeekMoE (`2401.06066`)：重塑MoE架构**
    *   **解决的问题**：传统MoE（如GShard）存在专家不专的问题，限制了模型潜力的发挥。
    *   **核心贡献（超参数层面）**:
        1.  **细粒度专家分割**：在总参数量不变的情况下，将标准FFN大小的专家切分成更小的单元（如4倍），同时激活更多数量的专家。这大大增加了专家组合的灵活性。
        2.  **共享专家隔离**：设置一组始终激活的“共享专家”来学习通用知识，让其他“路由专家”能更专注于学习特定、非重叠的知识。
    *   **意义**：这套“精细分割+共享隔离”的超参数设计，构成了后续所有DeepSeek大模型的基础组件，旨在以更低的计算量（激活参数少）达到更优的性能。

*   **MLA (`2405.04434` 在DeepSeek-V2中提出)：革命性的注意力机制**
    *   **解决的问题**：标准MHA的KV Cache随序列长度线性增长，成为推理速度的巨大瓶颈。GQA/MQA等方案在减少缓存的同时会损害性能。
    *   **核心贡献（超参数层面）**:
        1.  **低秩KV联合压缩**：将Key和Value共同压缩到一个极低维度的潜在向量中，推理时只需缓存这个小向量和一小部分解耦的Key，使KV Cache减少了惊人的93%以上。
        2.  **解耦的旋转位置编码（RoPE）**：巧妙地解决了低秩压缩与RoPE位置编码不兼容的问题。
    *   **意义**：MLA以极小的KV Cache实现了比肩甚至超越MHA的性能，是DeepSeek模型实现高效推理、支持超长上下文的关键。

#### 2. 融合与规模化：走向极致性价比

拥有了强大的架构组件后，团队将它们融合，并辅以极致的工程优化，将规模推向极致。

*   **DeepSeek-V2 (`2405.04434`)：经济高效的旗舰**
    *   **融合**：将DeepSeekMoE和MLA首次结合，构建了一个236B总参、21B激活的模型。
    *   **工程**：文中详细介绍了HAI-LLM训练框架、通信优化、负载均衡策略（专家/设备/通信三级平衡损失）等，确保了稀疏模型的高效训练。

*   **DeepSeek-V3 (`2412.19437`)：工程优化与算法设计的典范**
    *   **算法**：摒弃了传统的辅助损失，首创**辅助损失免费的负载均衡策略**，在动态平衡专家负载的同时，避免了因平衡损失带来的性能退化。引入**多Token预测（MTP）** 目标，不仅提升了模型性能，还可以用于推测解码加速推理。
    *   **工程**：引入**FP8混合精度训练**，并设计了精密的计算-通信重叠管道（DualPipe），使得训练671B参数的超大模型仅需278.8万H800 GPU小时，训练成本极低。

#### 3. 强化学习与推理能力涌现

在强大的基座模型之上，团队探索了后训练的极限，特别是推理能力的习得。

*   **DeepSeekMath (`2402.03300`)：RL算法先驱**
    *   提出了**分组相对策略优化（GRPO）**，通过在问题输出的组内计算相对优势，省去了传统PPO中与策略模型等大的价值模型，极大地降低了强化学习的计算成本。这个算法成为了后续后训练的标准配置。

*   **DeepSeek-R1 (`2501.12948`)：纯RL的推理涌现**
    *   **R1-Zero惊艳亮相**：证明了即使完全跳过监督微调（SFT），仅通过GRPO强化学习和基于规则的奖励，模型就能**自主涌现出复杂的推理行为**，如自我反思、验证和探索不同解法，这被称为“Aha moment”。
    *   **R1的实用化改进**：针对R1-Zero可读性差、语言混杂等问题，引入了少量冷启动SFT数据和**多阶段训练**（SFT -> RL -> ...），最终打造出性能媲美OpenAI o1的顶尖推理模型。

#### 4. 新征程：架构、记忆与系统的深度革新

在V3/R1的基础上，团队继续从模型架构、存储机制和推理系统层面寻求突破。

*   **注意力机制进化 (DSA/CSA/HCA)**
    *   **DeepSeek Sparse Attention (DSA) (`2512.02556`)**：为突破长上下文的计算瓶颈，在V3.2中引入稀疏注意力，通过一个轻量级的“闪电索引器”预测应该关注哪些Token，只对选中的Token进行核心注意力计算，将计算复杂度从平方级降低，效率大幅提升。
    *   **DeepSeek-V4的混合注意力**：进一步发展出**压缩稀疏注意力（CSA）** 和**重度压缩注意力（HCA）**，通过压缩KV缓存序列长度并结合稀疏/密集注意力，在百万Token场景下，其计算量和KV缓存大小仅是V3.2的零头。

*   **记忆与连接的进化 (Engram, mHC)**
    *   **Engram条件记忆模块 (`2601.07372`)**：观察到语言建模中既有动态推理，也有对固定范式（如命名实体）的静态查找。它通过为非计算密集型知识提供基于N-gram哈希的稀疏查找表，将这部分任务从MoE计算中卸载，提升了参数利用效率，是对稀疏性架构的互补性补充。
    *   **流形约束超连接 (mHC) (`2512.24880`)**：对网络拓扑结构进行创新，通过将残差连接的权重矩阵约束在固定的几何结构内，增强了深层网络的信号传播稳定性和训练效率，以可控的成本提升了模型性能。

*   **推理系统的精细化设计 (DualPath)**
    *   **DualPath推理系统 (`2602.21548`)**：该论文重点关注推理服务的**Prefill-Decode分离式架构**，引入**双路径KV缓存加载机制**（既可以从存储系统直接加载缓存，也可以让空闲的解码引擎预加载后再通过RDMA传递给预填充引擎）。结合对模型执行通信和KV缓存传输的精细化流量隔离与调度，打破了存储网络带宽瓶颈，大幅提升了多轮对话和智能体工作负载下的吞吐量。
